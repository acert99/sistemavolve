## Diagnóstico do incidente (por que KPI/briefing não rodaram hoje)

**Causa raiz encontrada:** o `crontab` do `root` apontava para scripts em **`/opt/volve/scripts/*`**, mas o diretório **`/opt/volve/scripts` não existia** (foi apagado).

Efeito:
- Cron executava, mas falhava com `... /opt/volve/scripts/<script>.sh: not found`.
- Consequentemente, **KPI diário** e **rotinas dependentes desses scripts** não rodavam.

Observação importante:
- Existem endpoints no app em **`/api/cron/*`** (Next.js) que funcionam como “executores” quando chamados com `Authorization: Bearer <CRON_SECRET>`.
- Porém, o agendamento no VPS também chamava scripts locais (shell) — e não apenas os endpoints.

## Recomendações para não repetir o problema

### Recomendação A (preferida): tornar o agendamento rastreável

Migrar o agendamento para **GitHub Actions schedule** chamando os endpoints `/api/cron/*` com secrets do GitHub.

Prós:
- Auditável (histórico de runs)
- Não depende de arquivos soltos no VPS

Contras:
- Depende do GitHub

### Recomendação B: versionar scripts de `/opt/volve/scripts`

Criar um diretório no repo (ex.: `ops/vps-scripts/`) com:
- `volve-kpi-daily.sh`
- `volve-client-reports.sh`
- `volve-kpi-weekly-close.sh`
- `clickup-weekly-start-telegram.sh`
- etc.

E um `scripts/deploy_vps_automations.sh` idempotente que:
- recria `/opt/volve/scripts`
- copia scripts
- valida permissões
- valida crontab

### Recomendação C: endurecer contra “apagão”

- Criar check diário de “sanidade”:
  - alertar se `/opt/volve/scripts` não existir
  - alertar se algum script do crontab não estiver presente
  - alertar se `/opt/volve/.env.telegram` estiver ausente

## Checklist operacional (runbook rápido)

Quando “automação não rodou”:

1) `crontab -l` (root) — entradas existem?
2) arquivos alvo existem?
   - `ls -la /opt/volve/scripts`
3) logs:
   - `tail -n 200 /opt/volve/logs/volve-kpi-daily.log`
   - `tail -n 200 /var/log/volve-cron.log`
   - `tail -n 200 /var/log/volve-scheduled.log`
4) env:
   - existe `/opt/volve/.env`?
   - existe `/opt/volve/.env.telegram`?
